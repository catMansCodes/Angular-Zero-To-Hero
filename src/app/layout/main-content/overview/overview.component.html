1. Introduction to Angular

Angular is a TypeScript-based front-end framework developed and maintained by Google. It is used to build scalable, high-performance, single-page applications (SPAs).

Angular follows a component-based architecture, where the UI is broken into reusable and independent components. It provides built-in solutions for routing, forms, HTTP communication, dependency injection, and state management.

Angular is widely used for enterprise-level applications due to its strong structure, long-term support, and robust tooling.

2. Install Angular (Windows & macOS)

Angular requires Node.js and npm (Node Package Manager). Once Node.js is installed, Angular applications are created and managed using the Angular CLI.

  • Install Node.js (LTS version recommended)
  • Install Angular CLI using npm
  • Create and run Angular projects via CLI commands

The CLI ensures consistent project structure, optimized builds, and built-in development tooling such as live reload and testing support.

3. Angular File & Folder Structure

Angular projects follow a standardized folder structure generated by the CLI. This structure helps developers maintain consistency and scalability.

Key folders and files:

  • src/ – Main application source code
  • app/ – Components, services, modules
  • assets/ – Images, fonts, static files
  • environments/ – Environment-specific configs
  • angular.json – Angular workspace configuration

This structure promotes separation of concerns and easier collaboration in teams.

4. Angular CLI

The Angular Command Line Interface (CLI) is a powerful tool that simplifies development tasks.

CLI capabilities include:

  • Creating projects, components, services, guards
  • Running development server
  • Optimized production builds
  • Testing and linting

Using the CLI ensures best practices are followed automatically.

5. Third-Party Tools & Framework Setup

Angular applications often integrate third-party libraries to enhance UI and functionality.

Commonly used tools:

  • Bootstrap – Responsive UI components
  • Font Awesome – Icon library
  • Google Fonts – Custom typography

These tools can be installed via npm or included using CDN links, depending on project requirements.
