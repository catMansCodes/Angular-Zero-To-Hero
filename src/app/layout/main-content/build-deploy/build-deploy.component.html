1. Production builds

A production build emits minified bundles, applies tree shaking, and compiles templates ahead of time (AOT) so browsers download less JavaScript and parse templates faster.

Use ng build --configuration production (or your named production configuration in angular.json).

What you gain
  • Smaller payloads and fewer round-trips.
  • Better runtime performance after load.
  • Errors caught at build time via AOT.

2. Environment configuration

Keep API base URLs, feature flags, and analytics keys out of hard-coded strings. Modern setups often use build-time replacement or runtime config (JSON loaded at startup) depending on security and rotation needs.

Never commit secrets; inject them in CI/CD or your hosting provider’s secret store.

Typical goals
  • Separate dev, staging, and production endpoints.
  • Switch behavior without branching source code per environment.
  • Align with your team’s twelve-factor or platform conventions.

3. Deployment options

Angular’s output is static assets plus index.html—ideal for CDNs, object storage, or web servers with SPA fallback routing (serve index.html for unknown paths).

For APIs you usually deploy separately (serverless, containers, or traditional backends).

Common targets
  • Static hosts (Firebase Hosting, Netlify, Vercel, S3 + CloudFront, Azure Static Web Apps).
  • Containers (Docker + Kubernetes or cloud runtimes).
  • Traditional IIS / nginx / Apache with configured try_files or URL rewrite.

4. Server-side rendering (SSR)

With Angular SSR (Universal), the server renders the first HTML response so users and crawlers see content sooner; the client then hydrates into a full SPA.

Mind APIs that only run in the browser (window, localStorage)—guard them for server context.

Why teams add SSR
  • Faster first paint on slow networks.
  • Better SEO for indexable content.
  • Consistent previews for social sharing meta tags when paired with the right server.

5. Static site generation (SSG)

SSG bakes HTML at build time for known routes—great when content is stable or updated on a schedule. It pairs well with docs, marketing sites, and blogs.

Hybrid apps may mix SSG for public pages with client-side routing for authenticated areas.

Good fits
  • Documentation and landing pages.
  • Content that rarely changes per request.
  • Maximum CDN cache efficiency.
