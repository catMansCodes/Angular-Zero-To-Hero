1. CRUD with JSON Server

JSON Server is a lightweight mock backend for frontend development. It supports full REST-style CRUD while your real backend is still in progress.

Supported operations
  • Create (POST)
  • Read (GET)
  • Update (PUT / PATCH)
  • Delete (DELETE)

2. API call integration with HttpClient

Angular uses HttpClient to call APIs and receive typed responses via Observables. You can compose streams, manage errors, and render loading states in components.

  • Observable-based response handling
  • Loading + error + success state UX
  • Easy testability with HttpClientTestingModule

3. Resource API (resource / rxResource)

Angular resource APIs provide a declarative way to fetch and cache remote data while exposing loading and error state in a signal-friendly style.

  • Signal-based data access patterns
  • Less manual RxJS wiring in simple data flows
  • Clear state surface for templates

4. Browser storage choices

Session Storage

Per-tab temporary state cleared when tab closes.

Local Storage

Persistent key-value data for preferences and cache metadata.

Cookies

Sent with requests; useful for session/auth workflows.

5. Functional guards, resolvers & defer

Use functional guards/resolvers to keep route protection and preloading concise. Combine with @defer to postpone heavy UI until needed and improve first render performance.

  • Cleaner route-level logic
  • Better perceived performance on content-heavy pages
