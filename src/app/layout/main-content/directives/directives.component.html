1. Directives overview

Directives are instructions in Angular templates that change the DOM, appearance, or behavior. They extend HTML with Angular-specific syntax.

Three common categories
  • Structural — add/remove DOM (*ngIf, *ngFor, *ngSwitch)
  • Attribute — change element behavior or style (ngClass, ngStyle)
  • Custom — your own attribute or structural directives

2. Built-in structural directives

Classic syntax uses the * prefix (syntactic sugar for <ng-template>).

3. Attribute directives: ngClass & ngStyle

These directives change classes or inline styles from component state without structural changes.

ngClass

Toggle adds a Bootstrap-friendly background on this panel.

ngStyle

Adjustable font size ({{ emphasisSize() }}px).

4. Classic * syntax vs control flow

Modern Angular supports @if, @for, @switch, and @empty for often clearer templates and good performance.

  • Older tutorials use *ngIf / *ngFor; new code often prefers @if / @for.
  • Both approaches are valid; learn both to read any codebase.

5. Custom directives

Build reusable behaviors with @Directive:

  • Auto-focus, click-outside, copy-to-clipboard
  • Permission-based *appHasRole structural directives
  • Host listeners for hover, resize, or scroll

6. Styling in Angular

Use component styles, global styles, or attribute directives (ngClass / ngStyle) together with encapsulation to avoid leaking CSS.
