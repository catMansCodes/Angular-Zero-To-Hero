1. Component Architecture & Lifecycle Hooks

Angular applications are built using components. A component controls a part of the UI and consists of:

  • HTML template (view)
  • TypeScript class (logic)
  • CSS styles (presentation)

Angular components follow a lifecycle managed by the framework. Lifecycle hooks allow developers to execute logic at specific moments.

Common lifecycle hooks:
  • ngOnInit()
  • ngOnChanges()
  • ngAfterViewInit()
  • ngOnDestroy()

2. Template Syntax & Data Binding

Angular templates use a powerful syntax to bind data between the component class and the view.

  • Interpolation – Display values (bind data from ts file to html file)
  • Property binding – Bind DOM properties
  • Event binding – Listen to user actions

This binding system ensures the UI stays in sync with application state.

3. Event Handling & Two-Way Binding

Event binding allows Angular applications to react to user interactions such as clicks, keyboard input, and form submissions.

Two-way binding synchronizes data between the component and template using [(ngModel)] or signal-based approaches.

Use cases:
  • Forms
  • Search inputs
  • Dynamic UI updates

4. Component Communication (Input & Output)

Angular supports communication between parent and child components using decorators.

  • @Input() – Pass data from parent to child
  • @Output() – Emit events from child to parent

This pattern ensures components remain loosely coupled and reusable.

5. Standalone & Reusable Components

Standalone components remove the dependency on NgModules, simplifying application structure.

Reusable components encapsulate UI and logic so they can be used across multiple screens.

Benefits:
  • Cleaner architecture
  • Better performance
  • Easier maintenance

6. Counter Application

A counter application is a common beginner project that demonstrates:

  • State management
  • Event binding
  • Template updates

It is often used to introduce best practices for component structure.

7. Decorators & Control Flow

Decorators provide metadata that tells Angular how to process classes and properties.

  • @Component – Defines a component
  • @Input / @Output – Component communication
  • @if, @else, @for, @switch – Modern template control flow

The new control flow syntax improves readability and performance compared to older structural directives.
