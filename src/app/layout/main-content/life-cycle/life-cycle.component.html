1. What is the component lifecycle?

Every Angular component goes through a predictable sequence from creation to destruction. At each step, Angular calls optional lifecycle hook methods on your class so you can react: load data, react to input changes, query the DOM, or clean up subscriptions.

Typical uses: fetch data in ngOnInit, integrate with third-party widgets in ngAfterViewInit, unsubscribe in ngOnDestroy.

2. Order of lifecycle hooks (simplified)

After the class is constructed, Angular runs hooks in roughly this order:

  1. ngOnChanges — when an @Input() value changes (not first run only; also on updates).
  2. ngOnInit — once after first ngOnChanges; good place for one-time setup and initial data loads.
  3. ngDoCheck — every change detection run; use sparingly (can affect performance).
  4. ngAfterContentInit — after content projected with ng-content is initialized.
  5. ngAfterContentChecked — after every check of projected content.
  6. ngAfterViewInit — after the component’s view (and child views) are initialized; safe to use @ViewChild references.
  7. ngAfterViewChecked — after every view check.
  8. ngOnDestroy — once before the component is destroyed; release timers, subscriptions, and listeners.

You only implement the hooks you need; each must match the name Angular expects (for example ngOnInit, not onInit).

3. Constructor vs ngOnInit

  • Constructor — use only for lightweight setup and dependency injection. Avoid heavy work or HTTP calls here.
  • ngOnInit — runs after inputs are set for the first time; preferred for initialization logic and loading data.

4. Example: implementing common hooks

Implement interfaces from @angular/core (optional but helps TypeScript catch typos) and add the matching methods: 

import { Component, OnInit, OnDestroy, Input, OnChanges, SimpleChanges } from '@angular/core';

@Component({
  selector: 'app-user-card',
  templateUrl: './user-card.component.html',
})
export class UserCardComponent implements OnChanges, OnInit, OnDestroy {
  @Input() userId = '';

  ngOnChanges(changes: SimpleChanges): void {
    if (changes['userId']) {
      // React when userId input changes
    }
  }

  ngOnInit(): void {
    // One-time setup, e.g. load data using this.userId
  }

  ngOnDestroy(): void {
    // Unsubscribe, clear timers, detach listeners
  }
}
